1. 改版总览

本次重点是创作链路前端体验升级，底层业务逻辑整体复用；快速模式仍为 2 步，专业模式仍为 3 步，资产/主体库/音色库/语言/积分管理/个人中心均复用原逻辑承接。

2. 【创作】首页-功能说明

2.1 页面构成

原型占位：首页整体 建议替换为：首页全屏截图
首页原型截图占位
建议截图内容：标题区、快速/专业模式切换、对话式提示词输入框、参数工具条、灵感广场案例区。
可替换为：2.1 首页-页面构成截图
区域 内容 说明
顶部创作区 模式切换 + 对话式提示词输入框 + 发送按钮 + 字数统计 创作启动主入口，发送后直接进入工作台。
参数工具条 风格、模型、规格、主体、图片、剧本 参数先在首页配置，再整体带入工作台。

2.2 总体业务逻辑

  1. 用户进入首页，默认展示快速模式。
  2. 用户可在首页完成模式选择、提示词输入与参数配置。
  3. 系统实时校验提示词合法性，并同步更新字数和可生成状态。
  4. 若用户点击灵感广场“做同款”，系统自动将案例 prompt 回填到输入框。
  5. 点击发送后，当前模式、提示词、风格、模型、规格、主体/图片/剧本状态全部带入工作台。
  6. 进入工作台后默认进入生成态，不再停留首页。

2.3 逐功能说明

2.3.1 模式切换

原型占位：首页-模式切换 建议替换为：快速模式/专业模式按钮截图
模式切换截图占位
建议展示：首页顶部快速模式、专业模式按钮，强调当前选中态与切换后的参数差异。
功能说明用户可在首页切换快速模式和专业模式，决定后续创作流程、参数上限和工作台步骤结构。
用户操作点击“快速模式”或“专业模式”按钮进行切换。
交互切换后立即更新按钮高亮、输入框 placeholder、字数上限、剧本入口显隐、工作台步骤、模型选项、预计生成时长，并给出 toast 提示。
限制快速模式强调效率，专业模式强调控制；切换时内容保留，但专业切快速若超过上限会被自动截断。
前端首页与工作台模式状态共用一份 state；所有 `.studio-mode-btn` 同步 active 与 `aria-selected` 状态。
后端接收 mode 字段，决定走快速工作流还是专业工作流；不需要额外新增模式判断页。
校验规则快速模式提示词上限 100 字；专业模式提示词上限 1000 字；切换到快速模式时自动裁剪超长内容。
补充逻辑快速模式工作台为 2 步；专业模式工作台为 3 步；专业模式显示剧本上传入口，快速模式隐藏。
测试用例1. 快速→专业切换后剧本按钮出现；2. 专业→快速切换后超长提示词被截断；3. 模式切换后工作台步骤数量正确；4. ETA 与按钮文案联动正确。
埋点需求`homepage_mode_switch`：记录 from_mode、to_mode、prompt_length、has_script、has_image。

2.3.2 提示词输入

原型占位：首页-提示词输入 建议替换为：输入框红框截图
提示词输入截图占位
建议展示：对话式输入框、发送按钮、字数统计、参数工具条与下方灵感广场的相对位置。
可替换为：2.3.2 提示词输入截图
功能说明首页输入框是创作启动入口，用户输入创意后点击发送，直接进入工作台生成。
用户操作在输入框填写提示词，点击发送按钮进入工作台。
交互输入过程中实时刷新字数计数；首页与工作台输入框内容实时同步；禁用态按钮展示 title 提示原因。
限制提示词不能为空，且不能包含敏感词。
前端使用对话式输入框 + 发送按钮 + 字数计数；通过统一 `currentPromptText` 管理首页和工作台输入值。
后端接收 prompt 作为主生成内容，参与脚本拆解、镜头规划、素材生成与后续成片任务。
校验规则空值禁止；命中“血腥、暴力、色情、淫秽、违法、仇恨”等词时禁止生成。
补充逻辑点击发送前若未通过校验，停留首页并提示错误；通过后才进入工作台。
测试用例1. 空 prompt 禁止发送；2. 敏感词提示正确；3. 首页输入后工作台同步；4. 专业模式大于 100 字但小于 1000 字允许提交。
埋点需求`homepage_prompt_input`、`homepage_prompt_send`：记录 mode、prompt_length、validation_result、blocked_reason。

2.3.3 风格选择

功能说明用于选择画面整体风格，如日式动漫、国风3D、CG电影、美式动漫、玄幻修仙、皮克斯风、赛博朋克。
用户操作点击“风格”工具项，打开风格面板，选择具体风格。
交互风格面板中当前选中项高亮；点击后立即回填到首页工具条与工作台表单。
限制单次只能选择一个风格。
前端通过样式弹层展示多个风格卡片，支持 hover 与 click 选中。
后端生成任务接收 style 字段，用于约束视觉风格；逻辑复用现有风格参数承接。
校验规则无强校验，默认值为“日式动漫”。
补充逻辑风格切换不清空 prompt，也不影响上传图片和剧本状态。
测试用例1. 风格选中后标签正确回填；2. 切换模式后风格保持；3. 返回首页后风格不丢失。
埋点需求`homepage_style_select`：记录 style_name、mode。

2.3.4 模型选择

功能说明用于选择当前创作模式下可用模型。
用户操作首页展示模型标签，进入工作台后通过下拉选择模型。
交互模式切换时模型文案与工作台模型下拉同步切换。
限制快速模式仅 STORY LAIHUA 1.0；专业模式可选 LAIHUA3.0 / LAIHUA 2.0。
前端工作台模型 options 根据 currentMode 动态生成；若当前模型不在目标模式列表内则自动重置为首选项。
后端根据 model 决定调用不同模型链路；业务逻辑复用，不新增页面分支。
校验规则提交生成时 model 必须属于当前 mode 的可选集合。
补充逻辑首页主要做轻提示，真正的可切换入口以工作台下拉为准。
测试用例1. 快速模式只显示 STORY LAIHUA 1.0；2. 专业模式显示两个模型；3. 专业切快速时模型自动回退。
埋点需求`model_select`：记录 source_page、mode、model_name、previous_model。

2.3.5 规格选择（分辨率 / 比例 / 时长）

功能说明统一设置清晰度、视频比例、秒数，并在首页工具条中摘要展示。
用户操作点击规格入口打开规格弹层，选择 540p/720p/1080p、16:9/9:16，拖动时长滑杆。
交互修改后摘要实时更新为“分辨率 · 比例 · 时长”；工作台表单同步更新。
限制快速模式时长 4-8 秒；专业模式时长 4-90 秒。
前端规格弹层中分组控制 + 滑杆控制；切换模式时自动 clamp 非法时长。
后端接收 resolution、ratio、duration 作为生成任务参数；逻辑复用现有规格配置方案。
校验规则resolution 仅允许 540p/720p/1080p；ratio 仅允许 16:9/9:16；duration 必须在模式范围内。
补充逻辑首页设置与工作台设置共享状态，避免二次选择成本。
测试用例1. 快速模式 duration 最大值为 8；2. 专业模式切换后 duration 最大值变为 90；3. 首页修改后工作台同步。
埋点需求`spec_change`：记录 resolution、ratio、duration、mode、source_page。

2.3.6 主体选择

功能说明用于打开主体库，选择角色模板或已保存主体，并将主体应用到当前草稿。
用户操作点击“主体”入口查看主体库内容。
交互支持 hover 预览、click 固定弹层、点击外部关闭；侧边栏主体库弹层动作支持“回填当前主体”。
限制生成中建议仅查看，不建议替换主体。
前端主体入口与侧边栏主体库复用同一承接逻辑；弹层支持固定与关闭状态控制。
后端主体选择结果以 subject_id / subject_meta 形式回填当前 draft；无新增底层流程。
校验规则无必填校验；若未选择主体则按默认逻辑生成。
补充逻辑主体回填后会沿用到分镜与视频生成阶段，保持角色一致性。
测试用例1. 点击主体库可打开；2. 回填成功后 toast 正确；3. 返回首页主体状态保留。
埋点需求`subject_library_open`、`subject_apply`：记录 source_page、mode、subject_id。

2.3.7 上传图片

功能说明用户可上传 1-4 张图片作为参考图，辅助生成角色、场景或构图。
用户操作点击“图片”按钮，选择本地图片。
交互上传成功后显示“已上传X张”；再次上传按覆盖/替换处理。
限制最多 4 张；单张最大 10MB；仅支持 JPG/PNG/WEBP；分辨率至少 512×512。
前端读取本地文件后先做格式、数量、大小与分辨率校验，校验失败保留旧状态并提示。
后端接收图片资源并挂接到当前草稿；业务逻辑复用现有参考图上传能力。
校验规则数量 > 4 拦截；格式非法拦截；大小超限拦截；分辨率不足拦截。
补充逻辑图片上传不影响主体库逻辑，二者并行存在。
测试用例1. 上传 5 张图片被拦截；2. 上传 11MB 图片被拦截；3. 上传成功后徽标正确；4. 再次上传能替换原状态。
埋点需求`image_upload_click`、`image_upload_success`、`image_upload_fail`：记录 count、size、fail_reason、mode。

2.3.8 上传剧本

功能说明专业模式下新增剧本上传入口，用于承接长文本剧情创作。
用户操作切到专业模式后点击“剧本”按钮上传文件。
交互上传成功后显示已上传标识，并通过 title 展示文件名；失败时 toast 提示。
限制仅专业模式显示；支持 txt / md / docx；文件大小不超过 10MB；空文件不允许。
前端脚本上传入口默认隐藏，专业模式时显示；文件先本地校验再进入提交流程。
后端接收 script 文件后完成解析并产出剧情摘要或镜头拆解中间结果。
校验规则扩展名白名单校验 + size 校验 + 空文件校验。
补充逻辑切回快速模式后剧本入口隐藏，但已上传状态不应直接丢失，回到专业模式可继续看到。
测试用例1. 快速模式不显示剧本入口；2. 上传 docx 成功；3. 上传 pdf 失败；4. 空文件失败。
埋点需求`script_upload_click`、`script_upload_success`、`script_upload_fail`：记录 file_ext、file_size、mode。

2.3.9 灵感广场

功能说明首页新增案例内容区，帮助用户浏览样例、筛选内容来源、快速复用 prompt。
用户操作点击筛选标签切换案例；悬停卡片查看“做同款”。
交互筛选后展示数量实时变化；卡片 hover 显示 CTA；点击做同款后 prompt 自动带入输入框。
限制当前原型展示 8 个样例，筛选分类包括全部 / 平台精选 / 热门趋势 / 官方模板 / 用户精选 / 行业场景。
前端通过 data-source 做本地过滤；不跳页、不刷新；仅更新当前展示列表与结果数量。
后端正式版建议通过案例接口返回 source、cover、title、prompt、热度等字段；当前原型为静态数据。
校验规则无强校验；若样例缺少 prompt，不允许执行做同款。
补充逻辑做同款只回填 prompt，不直接自动生成，仍由用户点击发送启动工作台。
测试用例1. 点击筛选后数量变化正确；2. 做同款后 prompt 正确回填；3. 做同款后输入框焦点正确落在末尾。
埋点需求`inspiration_filter_click`、`inspiration_card_click`、`inspiration_make_same`：记录 filter_type、card_id、source。

2.3.10 发送进入工作台

功能说明点击首页发送按钮后，直接从首页进入工作台，并默认进入生成态。
用户操作在完成提示词与基础参数设置后点击发送按钮。
交互首页淡出、工作台淡入；当前模式和草稿全部保留；进入后左侧先展示生成进度面板。
限制仅在 prompt 合法时允许进入。
前端通过隐藏/显示 landing-view 与 workbench-view 切换页面状态，不新开页面。
后端正式版建议先创建草稿/任务，再进入工作台；当前原型以本地状态模拟承接。
校验规则复用 prompt 校验规则。
补充逻辑进入工作台后自动调用 openWorkbenchGeneration，直接展示左侧进度状态。
测试用例1. 发送成功后直接进入工作台；2. 当前 mode 保持一致；3. 左侧不是编辑态而是生成态；4. 右侧步骤正确。
埋点需求`homepage_enter_workbench`：记录 mode、prompt_length、has_image、has_script、style、model、resolution、ratio、duration。
